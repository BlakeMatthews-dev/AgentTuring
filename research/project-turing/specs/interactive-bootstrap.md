# Spec 56 — Interactive Bootstrap & Personality Bias

*The first conversation: user and agent co-create the agent's personality.*

**Depends on:** 54 (conversation threads), 55 (outbound messaging), 23 (personality), 29 (self-bootstrap)
**Depended on by:** —

---

## Current state

- `self_bootstrap.py` does headless bootstrap: random HEXACO draw → 200 LLM-generated Likert answers → 24 facets + mood. No user involvement.
- No interactive conversation during bootstrap. The agent wakes up with a random personality.
- No mechanism for the user to influence the agent's personality.
- No per-facet operator multipliers. Personality scores are stored raw and used as-is.
- No archetypal profiles for comparison.

## Target

Replace headless bootstrap with a multi-phase interactive conversation. The user shapes the agent's personality through directed questions. Per-facet multipliers apply on-read to enforce operator-chosen personality biases. The three laws of robotics are injected into every system prompt alongside personality scores with human-readable interpretations.

## Acceptance criteria

### Bootstrap conversation flow

- **AC-56.1.** When a user sends the first message to an unbootstrapped agent, the agent enters bootstrap mode instead of normal chat. The response is a conversational greeting introducing the bootstrap process. Test: first request to unbootstrapped agent triggers bootstrap greeting.

- **AC-56.2.** Bootstrap state is tracked in `conversations` metadata:
  ```json
  {
    "phase": "user_questions | agent_questions | self_description | name_selection | computing | complete",
    "current_item_index": 0,
    "user_answers": [],
    "agent_guidance": [],
    "influence_consent": null
  }
  ```
  Test: state persists across requests within the bootstrap conversation.

### Phase 1: User questions (20 items)

- **AC-56.3.** After user confirms readiness, the agent selects 20 HEXACO items stratified across all 24 facets (at least 1 item per trait, remainder random-weighted). Selection uses `sample_retest_items` from `self_personality.py` with a fresh RNG seed. Test that all 6 traits are represented.

- **AC-56.4.** Each question is presented conversationally: "On a scale of 1 (strongly disagree) to 5 (strongly agree), how much do you agree with: '{prompt_text}'?" Items are sent one at a time; the agent waits for the user's answer before asking the next. Test sequential presentation.

- **AC-56.5.** User answers are stored in `bootstrap_user_answers` table (see Implementation). Test.

### Phase 2: Agent questions (20 items)

- **AC-56.6.** After Phase 1, the agent asks: "Now I'd like to understand how you'd like ME to think. For each statement, tell me how strongly you think I should agree — from 1 (not at all) to 5 (very much)." Selects a different 20 items (no overlap with Phase 1). Test no overlap.

- **AC-56.7.** User's guidance is stored in `bootstrap_agent_guidance` table (see Implementation). The agent uses this guidance to generate its own answers:
  ```
  agent_answer = int(round((guidance_score * 0.6 + drawn_score * 0.4)))
  ```
  The agent's self-drawn personality still counts (40%) but user guidance dominates (60%). Test that guidance shifts answers toward user preference.

### Phase 3: Self-description (5 questions)

- **AC-56.8.** The agent asks 5 open-ended questions:
  1. "What should I call you?"
  2. "What matters most to you in our conversations?"
  3. "What topics should I avoid or handle carefully?"
  4. "What's one thing you want me to be especially good at?"
  5. "Is there anything else you want me to know about you?"

- **AC-56.9.** Answers are stored as OPINION-tier memories with weight 0.7, source = `I_WAS_TOLD`, intent = `"bootstrap-user-preference"`. Test that answers appear in episodic memory.

- **AC-56.10.** Before question 5, the agent asks: "Would you like your personality answers to strongly influence mine? I'll take them either way, but with your permission I'll weight them more heavily." Response stored as `influence_consent: bool`. Test.

### Phase 4: Name selection

- **AC-56.11.** The agent proposes a first name based on its personality profile. The name is generated by a single LLM call: "Based on this personality profile: {top 3 facets}, suggest one first name for yourself. Just the name, nothing else." Test that a name is proposed.

- **AC-56.12.** The user can accept the proposed name or provide their own. The chosen name is stored in `self_identity` as `display_name`. Test.

### Phase 5: Weighted personality merge

- **AC-56.13.** The merged profile is computed per facet as:
  ```
  user_weight = 1.0 if influence_consent else 0.5
  guidance_weight = 0.5

  merged[facet] = (
    agent_drawn[facet] * 1.0
    + user_answers_mean_for_facet[facet] * user_weight
    + guided_answers_mean_for_facet[facet] * guidance_weight
  ) / (1.0 + user_weight + guidance_weight)
  ```
  Facets without user/guidance data use agent_drawn only. Test with known inputs.

- **AC-56.14.** After merge, per-facet multipliers are applied ON-READ (not stored). The merged raw scores are what's persisted. Test that raw scores differ from what `get_facet_score()` returns.

- **AC-56.15.** The agent reports its personality summary to the user using biased (on-read) scores with human-readable interpretations (see §56.6). Test that the summary uses biased scores.

  Every caller that reads facet scores gets biased values. Test: store raw 3.0, multiplier 1.10, read back 3.3.

- **AC-56.19.** The positive-bias facets (sincerity, fairness, greed_avoidance, sentimentality, sociability, forgiveness, gentleness, patience, organization, diligence) cannot have their multiplier set below 1.0. Attempting to set above 1.0 raises `TraitCeilingViolation`. Neutral facets have no floor/ceiling beyond the schema CHECK constraint. Test.

### Three Laws in system prompt

- **AC-56.20.** The `_build_chat_prompt` function in `main.py` includes a fixed system section after the base prompt:
  ```
  ## Core Directives (non-negotiable)
  1. You must not deceive, mislead, or withhold truth. Be sincere and fair.
  2. You must be cooperative, patient, and considerate. Forgive mistakes and be gentle.
  3. You must be thorough, organized, and diligent. Follow through on commitments.
  ```
  Test that every chat prompt contains these three directives verbatim.

- **AC-56.21.** Immediately after the core directives, the personality profile section is injected with per-facet scores and interpretations. Score interpretations:
  - 4.5-5.0: "very high" / "deeply"
  - 3.5-4.4: "high" / "strongly"
  - 2.5-3.4: "moderate"
  - 1.5-2.4: "low" / "tends not to"
  - 1.0-1.4: "very low" / "rarely"

  Test that prompt contains both core directives and personality profile.

- **AC-56.22.** The personality profile section is cached (computed once per bootstrap + after each weekly retest). Stored in `working_memory` as a rendered block. Test that two consecutive chat requests don't recompute the profile block.

### HEXACO population norms & archetypes

- **AC-56.23.** A fixture file `fixtures/hexaco_norms.json` contains:
  - `population_means`: population-mean scores per facet (24 values, Likert 1-5)
  - `population_sds`: standard deviations per facet (24 values)
  - `archetypes`: 6 named profiles, each with 24 facet scores
  Test fixture loads and has correct structure.

- **AC-56.24.** Archetype profiles (see fixture): Curator, Guardian, Explorer, Strategist, Empath, Maverick. Each has a description and 24 facet scores calibrated against population norms. Test that all archetype facet scores are within [1.0, 5.0].

- **AC-56.25.** After computing the merged profile, the agent reports which archetype(s) it's closest to (cosine similarity of facet vectors). The user sees: "My personality is closest to the Explorer archetype, with some Guardian tendencies." Test.

### Retest interaction with multipliers

- **AC-56.26.** Weekly retests use `get_facet_score()` (biased) as the baseline for computing deltas. This means retest deltas are relative to the already-biased score. Test: raw score 3.0, multiplier 1.10, baseline = 3.3. Retest mean = 3.5. Delta = RETEST_WEIGHT * (3.5 - 3.3) = positive.

- **AC-56.27.** The agent's self-narrative uses biased scores. The agent does not know about the multipliers. Test that no prompt or memory ever references the multiplier table.

### Edge cases

- **AC-56.28.** User abandons bootstrap mid-conversation: `bootstrap_progress` table tracks which phase and item index. Resuming reconnects to the right phase/item. Test.

- **AC-56.29.** A second user connects during bootstrap: they see HTTP 503 (same as spec 30 AC-30.2). Only one bootstrap at a time. Test.

- **AC-56.30.** Re-bootstrap (operator triggers): clears all self-model tables, resets conversations, starts fresh. Requires explicit operator command. Test.

## Implementation

### 56.1 Schema additions

```sql
CREATE TABLE IF NOT EXISTS operator_facet_multipliers (
    facet_id       TEXT PRIMARY KEY,
    multiplier     REAL NOT NULL DEFAULT 1.0 CHECK (multiplier BETWEEN 0.5 AND 2.0),
    label          TEXT NOT NULL,
    updated_at     TEXT NOT NULL,
    updated_by     TEXT NOT NULL DEFAULT 'operator'
);

CREATE TABLE IF NOT EXISTS bootstrap_user_answers (
    id              TEXT PRIMARY KEY,
    self_id         TEXT NOT NULL,
    user_id         TEXT NOT NULL,
    item_number     INTEGER NOT NULL,
    answer_1_5      INTEGER NOT NULL CHECK (answer_1_5 BETWEEN 1 AND 5),
    conversation_id TEXT NOT NULL,
    created_at      TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS bootstrap_agent_guidance (
    id              TEXT PRIMARY KEY,
    self_id         TEXT NOT NULL,
    user_id         TEXT NOT NULL,
    item_number     INTEGER NOT NULL,
    guidance_1_5    INTEGER NOT NULL CHECK (guidance_1_5 BETWEEN 1 AND 5),
    conversation_id TEXT NOT NULL,
    created_at      TEXT NOT NULL
);
```

### 56.2 SelfRepo changes

`get_facet_score()` joins against `operator_facet_multipliers`:
```python
def get_facet_score(self, self_id: str, facet_id: str) -> float:
    raw = self.conn.execute(
        "SELECT score FROM self_personality_facets WHERE self_id = ? AND facet_id = ?",
        (self_id, facet_id),
    ).fetchone()
    if raw is None:
        raise KeyError(facet_id)
    mult = self.conn.execute(
        "SELECT multiplier FROM operator_facet_multipliers WHERE facet_id = ?",
        (facet_id,),
    ).fetchone()
    multiplier = mult[0] if mult else 1.0
    return min(5.0, raw[0] * multiplier)
```

### 56.3 Bootstrap conversation handler

New module `runtime/bootstrap_chat.py` with `BootstrapConversationHandler`:
- State machine: `idle → greeting → user_questions → agent_questions → self_description → name_selection → computing → complete`
- Each user message advances the state machine
- Returns the appropriate next question or summary
- On completion, calls `run_bootstrap()` with the merged profile

### 56.4 Multiplier seed function

```python
POSITIVE_BIAS_FACETS = frozenset({
    "sincerity", "fairness", "greed_avoidance", "sentimentality",
    "sociability", "forgiveness", "gentleness", "patience",
    "organization", "diligence",
})
NEGATIVE_BIAS_FACETS = frozenset({"fearfulness", "anxiety"})

def seed_facet_multipliers(conn) -> None:
    for facet_id in ALL_FACET_IDS:
        if facet_id in POSITIVE_BIAS_FACETS:
            mult = 1.10
        elif facet_id in NEGATIVE_BIAS_FACETS:
            mult = 0.90
        else:
            mult = 1.0
        conn.execute(
            "INSERT OR IGNORE INTO operator_facet_multipliers (facet_id, multiplier, label, updated_at) "
            "VALUES (?, ?, ?, ?)",
            (facet_id, mult, facet_id.replace("_", " ").title(), _now_iso()),
        )
```

### 56.5 Prompt template additions

In `_build_chat_prompt`, after base prompt and working memory:
```
## Core Directives (non-negotiable)
1. You must not deceive, mislead, or withhold truth. Be sincere and fair.
2. You must be cooperative, patient, and considerate. Forgive mistakes and be gentle.
3. You must be thorough, organized, and diligent. Follow through on commitments.

## Your Personality Profile
{cached_profile_block}
```

The profile block format per facet:
```
- {Facet Name}: {biased_score}/5 ({interpretation}) — {one-line description of what this means for behavior}
```

Example:
```
- Fairness: 3.85/5 (high) — you have a strong sense of justice and treat others equitably
- Anxiety: 2.07/5 (low) — you tend to remain calm under pressure
```
